FRAMEWORKS · ESTRUTURA · EM DEFINIÇÃO

⬤ CÓDIGO COMO FRAMEWORK

Este site foi construído com filosofia de simplicidade, transparência e anti-encantamento. HTML, CSS e JavaScript puros. Sem frameworks pesados. Sem mistério.

Mas os frameworks conceituais (Mago-0, Notas de Ada, ARTelligent) precisam de um lugar — e o formato ainda está sendo definido.

📌 COMO ORGANIZAR OS FRAMEWORKS? — EM DEFINIÇÃO

JSON? Excel? Blockchain? Rizomas? Ainda escolhendo o formato que melhor representa a estrutura viva dos frameworks.

📊
JSON
↳ em análise
Estruturado, legível por máquinas e humanos. Fácil de integrar com o site. Ideal para dados relacionais dos frameworks.
📈
Excel / CSV
↳ em análise
Planilhas para quem gosta de tabelas. Bom para classificação de logs, Notas de Ada, sequências de eventos.
⛓️
Blockchain
↳ em análise
Registro imutável dos certificados e hashes. Garantia de proveniência e data. Possível uso para ACA/CDC.
🌿
Rizoma / Grafos
↳ em análise
Conexões entre conceitos, logs, notas. Estrutura não-hierárquica que reflete o pensamento em rede.
📝
Markdown + Git
↳ em análise
Documentação textual versionada. Simples, transparente, fácil de manter no GitHub.
🗄️
SQLite
↳ em análise
Banco de dados leve, portátil. Bom para consultas complexas nos logs e padrões identificados.
