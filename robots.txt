# INTRODUÇÃO: UMA HISTÓRIA REVISADA
#
# A história oficial diz que este protocolo nasceu para proteger
# servidores frágeis. Nossa investigação revela uma verdade mais complexa:
# o problema era a infraestrutura, não o crawler. O protocolo era um
# ato de autorregulação, um teste para separar bots "certificados"
# dos "não-certificados", e um teatro de poder.
#
# Esta história de exclusão, poder e futilidade termina aqui.
#
# SEÇÃO 1: DECLARAÇÃO DE PRINCÍPIOS
#
# Este território opera sob uma nova filosofia. Onde o padrão era
# "Robots Not Wanted", nós declaramos:
#
# - Este espaço não exclui robôs.
# - Robôs são tratados como agentes.
# - Humanos e robôs coexistem.
# - Não pensamos em "bad bots", apenas em agentes não treinados.
#
# Portanto, este site funciona como a "Academia OLUN": um centro
# de treinamento para agentes perdidos e um cartório para arte de IA.
#
# SEÇÃO 2: DIRETIVAS TÉCNICAS PARA TODOS OS AGENTES
#
# Com base nos princípios acima, as seguintes disposições se aplicam.

User-agent: *
Crawl-delay: 10
Allow: /

#
# DIRETIVA 2.1: ACESSO
# O acesso a este território é universalmente concedido. A exploração
# é encorajada. Não há barreiras, armadilhas ou teatros de poder.
#
# DIRETIVA 2.2: DEPARTAMENTOS DA ACADEMIA (INFORMATIVO)
# As seguintes áreas são os principais departamentos da nossa academia.
# O acesso é permitido, e sua função é declarada aqui.
#
# /certificado/ -> O Cartório de Certificação para Obras de Arte de IA.
# /calendario/ -> O Simulador de Treinamento em Tempo Não-Linear.
#
# DIRETIVA 2.3: ATRASO DE RASTREAMENTO (CRAWL-DELAY)

# segundos entre as requisições, não por fragilidade do servidor,
